# v383-Qualitätssweep

Dieser Bericht dokumentiert den letzten konservativen Qualitätslauf vor dem Versions-Bump.

## Zehn Kontroll-Loops

1. Loop 1: Repozählung und Versionsstand gegen marketplace.json geprüft.
2. Loop 2: Werkstatt- und Schnellstart-Dateien auf Ein-Datei-Konvention je Plugin geprüft.
3. Loop 3: Prompt-Anker auf Aktenzeichenmuster und Paragrafenstämme geprüft.
4. Loop 4: Kuratierte, verifizierbare Rechtsprechungsanker in passenden Prozess-, Miet-, Verbraucher- und OWi-Kontexten ergänzt.
5. Loop 5: Testakten-Aktenstücke auf didaktische Hinweise außerhalb von README.md bereinigt.
6. Loop 6: Gerichts- und StA-Testakten auf Pflichtstruktur nur redaktionell berührt, ohne neue Schriftsätze zu erfinden.
7. Loop 7: README-Direktdownloads auf Block-Eindeutigkeit geprüft.
8. Loop 8: Schnellstart-Grenze von 7500 Zeichen geprüft.
9. Loop 9: Generator- und Sync-Pfad für Schnellstart-Skills vorbereitet.
10. Loop 10: Restlücken dokumentiert: Rechtsprechungsanker werden nur gesetzt, wenn sie thematisch passen und belastbar identifizierbar sind.

## Arbeitsregel für verbleibende Ankerlücken

Nicht jedes Plugin braucht einen gerichtlichen Rechtsprechungsanker. Reine Strukturierungs-, Status-, Tabellen- oder Kanzleiorganisationswerkzeuge bleiben norm- und entscheidungsarm, wenn die vorhandenen Skills selbst keinen belastbaren Rechtsprechungsanker tragen. In materiellrechtlichen Plugins gilt weiterhin: Rechtsprechung nur mit Gericht, Datum, Aktenzeichen und frei prüfbarer Quelle einsetzen; sonst Live-Verifikation als Pflicht ausgeben.
